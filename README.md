# Разработка реляционной базы данных для агрегатора студенческих скидок и акций

## 📌 Общая информация
* **Предметная область:** Централизованная система агрегации специальных предложений, скидок и промо-акций для студентов.
* **Проблема:** Трудности с поиском актуальной информации о студенческих льготах из-за её разрозненности в открытых источниках, а также упущенная выгода бизнеса из-за слабого охвата студенческой аудитории.
* **Решение:** Проектирование комплексной базы данных, объединяющей верифицированных пользователей и бизнес-партнеров в рамках единой экосистемы.
* **Разработчики:** Команда студентов группы ББИ2406.

## 🛠 Технические особенности и архитектура
Проект представляет собой базу данных и ядро серверной бизнес-логики платформы, реализованные на базе СУБД PostgreSQL.

* **Проектирование схемы:** Разработана нормализованная реляционная структура данных, исключающая избыточность и аномалии обновления.
* **Целостность данных:** Использование строгой типизации, ограничений целостности и перечисляемых типов для обеспечения корректности данных на уровне СУБД.
* **Автоматизация бизнес-правил:** 
  * Реализован механизм автоматического управления жизненным циклом публикаций и архивации устаревших предложений по времени.
  * Внедрена концепция «мягкого удаления» для сохранения истории и аналитической ценности данных.
* **Аудит и безопасность:** Настроено автоматическое триггерное логирование изменений параметров акций в формате полуструктурированных данных (JSONB) для ведения сквозного аудита.
* **Исключение конфликтов:** Применены продвинутые механизмы контроля временных диапазонов, предотвращающие пересечение конкурирующих предложений для одного и того же заведения.
* **Аналитический слой:** Разработан пул пользовательских функций и представлений для генерации операционной отчетности, расчета операционных метрик активности и агрегации данных о поведении пользователей для последующего формирования персональных рекомендаций.
* **Валидация:** Стабильность архитектуры, триггеров и запросов подтверждена комплексным набором позитивных и негативных тестов.

## 🗄 Функциональные домены (Структура данных)
1. **Управление пользователями:** Учетные записи с гибкой ролевой моделью доступа, модулем верификации студенческого статуса и персональными настройками поиска.
2. **Профили партнеров:** База данных партнерских организаций с геоданными, категориями деятельности, расписанием работы и контактной информацией.
3. **Каталог предложений:** Ведение учета акций, классификация типов скидок, условий их получения и периодов действия.
4. **Взаимодействие и аналитика:** Модули отзывов и оценок, механизмы активации купонов и сбор логов активности для оценки вовлеченности аудитории.
